X SWIM TEAM API

API documentation

    Swimmer API, endpoint: http://local.IP.address:port_number/swimmer/<user_id>

    GET
      Request
      • Example: curl http://local.IP.address:port_number/swimmer/2
      • Use user_id=0 to list all swimmers and their associated IDs
      Response
        On success
        • Response status code will be 200
        • Response body contains the swimmer's info in JSON format
        On error
        • Response status code will be 404
        • Response body: "There is no user with the ID <user_id>!
    PUT
      Request
      • Required data: name (first and last), birthdate (yyyy-mm-dd format), swimmer_group, and main_event
      • Example: curl -X PUT -d "name=Sehajvir Singh" -d "birthdate=2001-03-03" -d "swimmer_group=HP" -d "main_event=Backstroke" http://local.IP.address:port_number/swimmer/4
      • This cURL format is possible with Flask's reqparse (see here for more info)
      Response
        On success
        • Response status code will be 201
        • Response body contains an updated list of swimmer names + IDs in JSON format
        On error
        • Response status code will be 409
        • Response body: "There is already a user with the ID <user_id>!
    PATCH
      Request
      • Optional data to update: name (first and last), swimmer_group, and main_event
      • Example: curl -X PATCH -d "main_event=Sprint free" http://local.IP.address:port_number/swimmer/2
      • This cURL format is possible with Flask's reqparse (see here for more info)
      Response
        On success
        • Response status code will be 202
        • Response body contains the updated swimmer info JSON format
        On error
        • Response status code will be 409
        • Response body: "There is already a user with the ID <user_id>!
    DELETE
      Request
      • Example: curl -X DELETE http://local.IP.address:port_number/swimmer/3
      Response
        On success
        • Response status code will be 204
        • Delete does not return JSON serializable data
        On error
        • Response status code will be 404
        • Response body: "There is no user with the ID <user_id>!

    Coach API, endpoint: http://local.IP.address:port_number/coach/<user_id>

    GET
      Request
      • Example: curl http://local.IP.address:port_number/coach/2
      • Use user_id=0 to list all coaches and their associated IDs
      Response
        On success
        • Response status code will be 200
        • Response body contains the coach's info in JSON format
        • Coaches' ages are not included
        On error
        • Response status code will be 404
        • Response body: "There is no user with the ID <user_id>!
    PUT
      Request
      • Required data: name (first and last), birthdate (yyyy-mm-dd format), and coach_group
      • Example: curl -X PUT -d "name=Siobhan Newell" -d "birthdate=1995-02-23" -d "coach_group=13%2b provincial" http://local.IP.address:port_number/coach/3
      • This cURL format is possible with Flask's reqparse (see here for more info)
      Response
        On success
        • Response status code will be 201
        • Response body contains an updated list of coach names + IDs in JSON format
        On error
        • Response status code will be 409
        • Response body: "There is already a user with the ID <user_id>!
    PATCH
      Request
      • Optional data to update: name (first and last), coach_group
      • Example: curl -X PATCH -d "coach_group=13%2b provincial" http://local.IP.address:port_number/coach/2
      • This cURL format is possible with Flask's reqparse (see here for more info)
      Response
        On success
        • Response status code will be 202
        • Response body contains the updated coach info JSON format
        On error
        • Response status code will be 409
        • Response body: "There is already a user with the ID <user_id>!
    DELETE
      Request
      • Example: curl -X DELETE http://local.IP.address:port_number/coach/2
      Response
        On success
        • Response status code will be 204
        • Delete does not return JSON serializable data
        On error
        • Response status code will be 404
        • Response body: "There is no user with the ID <user_id>!
